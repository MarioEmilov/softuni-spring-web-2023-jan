1. HTTP е протокол за пренос на информация между уеб сървър и клиент
2. HTTP response потокът от информация от уеб сървъра до клиента. Той е изграден от:
-  статус линия (в която се съдържа информация за версията на протокола, статус код и статус фраза), 
-  response headers (мета дата за ресурса), 
-  body/тяло (съдържанието на HTTP response)
3. HTTP request е заявката, която се изпраща от клиента до сървъра за да се достъпят желаните данни. Състои се от:  
- рекуест линия (в която се съдържа информация за рекуест метода "GET", "POST", "PUT", "DELETE"; 
- Request URL - URL на ресурса; 
- версията на HTTP протокола;
4. Най-изпозлваните HTTP методи са "GET", "POST", "PUT", "DELETE", "TRACE"
5. Статус кода е съобщение, което уеб сървъра изпраща до клиента/браузъра за да индикира дали съответната заявка може да бъде изпълнена
6. Статус кодовете са разделени в 5 класа, като най-често използваните са:
- 200 "OK";
- 301 "Moved Permanently"
- 404 "Not found"
- 500 "Internal Server Error"
- 503 "Service Unavailable"
7. Разликата м/у HTTP и HTTPS, е че при HTTPS връзката е секюрната
8. Основната разлика м/у HTTP1.1 и HTTP2  драстичната оптимизация на комуникация при по-новата версия.При него са представени няколко важни елемента
- Multiplexing - обработка на множество потоци от информация чрез една TCP конекция
- Header compression - де-дубликация и поддръжка на хедър таблица между рекуестите
- Server Push - процес по изпращане на ресурси до клиента, без той да ги е заявил изрично
